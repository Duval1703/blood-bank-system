OPTION A: SQLite (Default - Perfect for Testing/Demo)
──────────────────────────────────────────────────────────────────────────────
No additional setup required! Already configured.

Environment Variables:
    DB_CONNECTION=sqlite
    DB_DATABASE=/app/database/database.sqlite

Pros: ✓ Zero configuration  ✓ Fast deployment  ✓ Free
Cons: ⚠️ Data resets on each redeployment (not suitable for production)

OPTION B: PostgreSQL (Recommended for Production)
──────────────────────────────────────────────────────────────────────────────
Step 1: In Railway dashboard, click "+ New" → "Database" → "Add PostgreSQL"
Step 2: Railway automatically creates database variables
Step 3: Update your environment variables:

    DB_CONNECTION=pgsql
    DB_HOST=${{PGHOST}}
    DB_PORT=${{PGPORT}}
    DB_DATABASE=${{PGDATABASE}}
    DB_USERNAME=${{PGUSER}}
    DB_PASSWORD=${{PGPASSWORD}}

Pros: ✓ Persistent storage  ✓ Production-ready  ✓ Better performance
      ✓ Automatic backups (paid plans)

═══════════════════════════════════════════════════════════════════════════════
                      🔧 TROUBLESHOOTING GUIDE
═══════════════════════════════════════════════════════════════════════════════

ISSUE: White screen or blank page
SOLUTION: 
  - Check APP_KEY is set correctly in Railway variables
  - Temporarily set APP_DEBUG=true to see error details
  - Check Railway deployment logs for errors
  - Verify all environment variables are correct

ISSUE: 500 Internal Server Error
SOLUTION:
  - Ensure APP_KEY is generated and added to Railway
  - Check database connection settings
  - Review Laravel logs in Railway dashboard
  - Verify PHP version is 8.2 or higher

ISSUE: Database connection failed
SOLUTION:
  For SQLite: Ensure DB_CONNECTION=sqlite
  For PostgreSQL: Verify database service is running and variables are correct

ISSUE: Assets not loading (CSS/JS)
SOLUTION:
  - Check build logs for "npm run build" success
  - Verify APP_URL matches your Railway domain
  - Clear browser cache

ISSUE: Login not working
SOLUTION:
  - Verify database was seeded (check deployment logs)
  - Try resetting database: redeploy app
  - Check credentials are correct (email/password case-sensitive)

For more troubleshooting: See RAILWAY_DEPLOYMENT.md, section "Troubleshooting"
